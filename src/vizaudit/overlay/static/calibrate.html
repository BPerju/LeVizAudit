VizAudit calibration

Connects directly to a camera on this machine -- no dataset, no recording session, no Rerun. Camera view (left): drag the 4 corners onto the real workspace surface, mark extreme-reach points while moving the arm, or pick an Edit tool to mark regions points shouldn't spawn in. Orthographic view (right): the same surface undistorted, where the marked points get fit to a circle, and where cuts can also be edited directly -- drag either view's handles to fine-tune, and watch the other view update to match. Lock the corners (the padlock icon) once they're aligned, to avoid nudging them by accident while marking points or drawing cuts. While drawing a polygon cut: double-click, or the checkmark button, closes it; Esc cancels it, or click the active tool icon again to stop. Set "Steps" and a distribution, and the live preview always shows exactly that many points, confined to the circle, the workspace, and outside every cut, in both views.

If the browser refuses camera access on this page: vizaudit-calibrate serves it on http://localhost and opens your default browser automatically by default, which avoids the camera-permission restrictions some browsers apply to file:// pages (and that a VS Code/editor preview pane applies regardless of origin). If you opened this file directly some other way, run vizaudit-calibrate --output calibration.html without --no-serve instead, or manually serve it yourself, e.g. python -m http.server in this file's directory.
