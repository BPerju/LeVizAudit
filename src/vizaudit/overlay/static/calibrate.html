VizAudit calibration

Marks a robot's reachable workspace on a live camera feed and configures how target points are sampled within it, for guided data-collection.
How to: in Workspace setup -- 1) drag the 4 corners onto the work surface; 2) "Mark reach points" and click 3+ spots at the edges of where the arm can physically reach; 3) "Fit circle". Then tune Pattern/Rotation below and copy the YAML below.

Objects

Define the things this scene places (one per gripper, one per block, etc.). To place one: pick its chip below, then use the floating toolbar's "select" tool (box-drag or click) on either view to select sample points, then click "Assign" here -- "Unassign" removes the active object from the current selection. A point can hold several objects; their stacking order is figured out automatically from the order you assigned them, not something you set by hand -- the episode preview below draws a stack as one dot plus a small legend showing which object is on top. The knobs below are REAL config fields the live overlay reads, not just a local preview -- see CLAUDE.md.

Config snippet

If the browser refuses camera access on this page: vizaudit-calibrate serves it on http://localhost and opens your default browser automatically by default, which avoids the camera-permission restrictions some browsers apply to file:// pages (and that a VS Code/editor preview pane applies regardless of origin). If you opened this file directly some other way, run vizaudit-calibrate --output calibration.html without --no-serve instead, or manually serve it yourself, e.g. python -m http.server in this file's directory.
